# 更新日志

本文档记录 Aster 桌面客户端的版本更新内容。GitHub Release 会在构建时自动读取对应版本小节，并追加安装包下载说明。

## [Unreleased]

### 新增

- 系统设置新增“软件更新”面板，可查看当前版本、检查 GitHub Releases 最新版本、阅读更新说明、下载并安装更新包，安装完成后可直接重启客户端完成更新。
- 客户端启动后会静默检查一次更新；检测到新版本时在主界面提示，未发现新版本时不打扰日常登录和业务操作。
- 接入 Tauri 2 官方在线更新能力，GitHub Release 会随安装包发布 `latest.json` 更新清单和 Windows/macOS updater 签名资产。
- README 改为中文主文档，并保留独立英文版本入口，方便中文交付和仓库首页阅读。

### 优化

- 在线更新安装前会按运行模式处理数据保护：主机/单机模式自动创建一次本地手动备份，客户端模式不直接操作主机 SQLite 数据库，直接安装客户端更新。
- GitHub Actions 发布流程会把安装包、updater 包、签名文件和 `latest.json` 一起上传到 Release，并将 `latest.json` 写入发布说明下载表。
- 优化 README 顶部产品定位文案，改为更贴近酒店物资流转、库存成本管理和本地化部署场景的企业级表达。

## [0.1.6] - 2026-07-08

### 新增

- 物品档案列表新增表格工具条，在筛选区和表格之间集中放置表格级操作。
- 物品档案新增导入入口，可直接选择 Excel 并按“只导入物品档案”模式导入。
- 物品档案新增导出能力，可按当前筛选条件导出 Excel；客户端模式下会导出到当前电脑自己的默认导出目录。
- 登录页新增“连接主电脑”入口，新电脑可先完成客户端连接和配对，再使用主机上的子账号登录。
- 找回密码改为独立子窗口流程，登录页只保留入口，验证码发送和密码重置在子窗口内完成。
- 登录页新增“记住账号密码”选项，可在当前电脑保存登录账号和密码，取消勾选会清除本机保存内容。

### 优化

- 物品档案筛选区与近期表头搜索样式保持一致，搜索表单和操作按钮采用上下布局，搜索按钮靠左。
- 连接向导支持登录前客户端专用流程，仅展示搜索主机、手动输入地址和输入配对码，主机共享管理仍需管理员登录。
- 优化找回密码表单样式，复用编辑器窗口布局，避免登录卡片展开后内容拥挤。
- 优化登录页操作布局，将“记住账号密码”和“找回密码”放在同一行，减少登录卡片按钮堆叠。
- 优化多电脑连接向导布局，收紧窗口尺寸，重排搜索主电脑列表和底部操作区，减少大块留白。

### 修复

- 修复操作日志 `entity_id` 为空时列表读取报错的问题，兼容旧审计记录和系统自动重建库存余额记录。
- 修复入库/出库单详情窗口中商品明细和批次成本明细表格相互挤压、表头覆盖内容的问题。

## [0.1.5] - 2026-07-07

### 新增

- 新增入库采购批次能力：确认入库后按每条明细生成独立采购批次，保留实际进货价、原始数量、剩余数量和剩余金额。
- 新增批次成本追溯：单据详情可查看批次成本明细，库存台账可打开批次库存窗口查看来源单据、供应商、批次单价和剩余金额。
- 新增物品“参考售价”，与“参考进价”分离；参考进价用于采购预填，参考售价用于酒店客人销售预填。
- 新增客人销售毛利核算：客销出库同时记录销售收入、FIFO 成本和毛利，报表中心与 Excel 导出新增销售毛利数据。
- 新增员工使用手册，提供 Markdown 和可本地双击打开的 HTML 两种格式，按业务功能说明物品档案、入库、出库、台账、流水、盘点、报表、备份和多电脑连接的日常使用方式。
- 新增新版 Excel 导入模板导出能力，模板固定为“物品档案 / 入库明细 / 出库明细”三张表，方便按真实业务单据导入。

### 优化

- Excel 导入不再按旧酒店月报或旧通用模板推算库存；入库明细会生成真实入库单和采购批次，出库明细会按内部领用或酒店客人销售生成真实出库单。
- Excel 导入复用正常出入库提交逻辑，出库按 FIFO 扣减批次并写入实际成本，客人销售同时保留销售收入、成本和毛利。
- 导入模板要求业务时间包含真实时分秒，避免只按月份或日期导入导致库存流水顺序、批次成本和报表期间不准确。
- 入库列表和出库/领用列表新增“物品”摘要列，单据批次可直接看到包含的商品，并支持按商品编码、名称和规格搜索对应单据。
- 入库列表和出库/领用列表操作列新增“详情”按钮，可打开只读子窗口查看单据头信息和商品明细。
- 入库、出库/领用、库存台账、库存流水和报表中心的商品筛选统一改为模糊搜索下拉，支持“全部物品”并按编码、条码、名称、规格和单位快速定位。
- 库存盘点页简化为盘点记录列表，详情改为独立子窗口展示，避免“盘点明细”在主页面重复分组；详情窗口内可查看统计、商品明细并执行导出、录入实盘、确认和作废操作。
- 系统设置“业务与目录设置”支持客户端模式打开；客户端仅保存当前电脑的默认导出目录和默认备份目录，业务参数继续由主机统一控制。
- 物品档案列表移除容易误解的价格列，编辑页保留“参考进价”仅作为入库录入预填；真实库存成本以入库批次为准。
- 入库、内部领用和酒店客人销售的金额语义完成拆分：入库记录采购价/采购金额，内部领用记录 FIFO 实际成本，客人销售同时记录销售收入、FIFO 成本和毛利。
- 客人销售允许低于成本价提交，但报表会单独标记负毛利，避免采购成本、销售收入和消耗成本混算。
- 单据详情按业务类型展示采购金额、成本金额、销售金额和毛利，客人销售可直接核对收入、成本和毛利。
- 出库、调整损耗和盘点盘亏按 FIFO 自动扣减批次，跨批次出库会拆分批次流水并回写单据实际成本。
- 单据作废支持批次冲正：出库作废会恢复被消耗批次，已被后续出库消耗的入库批次会阻止直接作废。
- 兼容旧测试/历史余额数据，现有库存余额会在首次批次出库或迁移时生成期初库存批次。
- 入库、出库、调整和盘点的业务日期改为真实日期时间录入并落库，后端统一拒绝晚于当前时间的业务时间；旧的仅日期历史记录会按已有创建时间补齐到日期时间。
- 报表和导出涉及日期筛选时按完整日期时间范围查询，避免当天业务记录因时间部分被遗漏。

### 修复

- 修复商品模糊搜索下拉面板在表格页中定位偏移的问题，改为挂载到页面顶层并按输入框真实坐标对齐。
- 修复入库和出库/领用列表默认显示当前月份但首次点击搜索未按当前月筛选的问题。

## [0.1.4] - 2026-07-05

### 优化

- 预算控制支持按“部门 + 月份”设置总预算，分类改为可选；选择分类时继续作为分类预算，不选择分类时作为部门月度总预算。
- 内部员工领用会先校验部门总预算，再校验分类预算；酒店客人销售继续只扣减库存，不占用部门预算。

## [0.1.3] - 2026-07-03

### 新增

- 主机模式“已连接的其他电脑”新增移除能力，可由主机管理员踢掉指定客户端设备。
- 出库单新增“出库类型”，支持区分内部员工领用和酒店客人销售；客人销售会扣减库存但不占用部门易耗品预算。

### 优化

- 移除客户端会同步删除主机端配对凭据、清理运行时连接缓存并写入操作日志，被移除设备需要重新配对后才能访问主机数据。
- 入库和出库单据明细的物品选择改为模糊搜索下拉，支持按编码、条码、名称、规格、单位等信息快速筛选并回填物品。
- 出库/领用单据明细新增“可用库存”列，选择物品后直接显示当前库存数量，零库存以警示样式标记。
- 优化报表中心页面布局，按筛选导出、经营分析、库存总览、领用分析、流水明细、库存风险分组展示，并在左侧导航新增“报表分析”分组。
- 报表中心移除页面内表格 list，改为指标、图表和摘要卡片展示，完整明细保留在 Excel 导出中。
- 报表中心筛选去掉与月份重复的开始/结束日期入口，统一按月份、部门、分类、物品和供应商查询。
- 月份字段统一改为年份和月份下拉选择，避免原生月份输入在桌面 WebView 中表现为可编辑文本。

### 验收说明

- 本轮已通过 `npm run build`、`cargo check`、`cargo test stock_repository` 和 `git diff --check` 验证；CI 构建时会继续执行发布验证并生成 release evidence。

## [0.1.0] - 2026-07-02

### 新增

- 完成 Aster Windows 与 macOS 桌面客户端全量功能交付，技术栈为 Tauri 2、React、Rust 和 SQLite。
- 支持本地 SQLite 数据库、库存入库/出库/调拨/调整/作废/冲销闭环、盘点、报表、Excel 导入导出、用户角色权限、预算规则和审批。
- 支持局域网单主机多客户端同步模式，主机持有唯一 SQLite 数据库，客户端通过主机 API 访问，不共享数据库文件。
- 支持本地冗灾能力，包括自动备份、手动备份、第二备份目录、导入前备份、恢复前备份、恢复校验 token、失败回滚和保留策略。
- 新增 Windows/macOS 双端 GitHub Actions 构建流程，Windows 安装包由 GitHub Windows runner 构建，macOS 安装包由 GitHub macOS runner 构建。

### 优化

- 新增“多电脑连接”向导，将主电脑开启共享、局域网搜索、手动输入地址、输入配对码和保存连接凭据合并为连续流程，减少普通用户理解主机/客户端、IP、端口和配对步骤的成本。
- 优化设置页“多电脑连接”和“备份与恢复”布局，使用紧凑状态区、摘要信息格和收敛提示条展示连接状态、配对码、备份路径和错误信息，避免大屏横向信息分散。
- 修复未完成主机配对时无法打开连接向导的问题，向导初始化不再提前调用需要主机 token 的系统设置接口。

### 验收说明

- 自动构建、发布校验、覆盖扫描和验收交接脚本已接入。
- 已补充 Windows 主机模式配对、macOS 客户端写入、主机重启后 token 持久性、SQLite 离线数据和自动备份包结构的实测证据。
- 最终业务验收仍需补齐 Windows/macOS 真实安装、首次启动、登录建库、Excel 导入导出、备份恢复和双机互联证据。
