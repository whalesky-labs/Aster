# Aster 冗灾与主机故障切换操作文档

## 1. 适用场景

本文件用于 Aster 在单机模式或局域网主机模式下发生故障时恢复业务。

适用情况：

- 主机电脑损坏或无法开机。
- 主机 SQLite 数据库损坏。
- 误导入、误恢复或误操作后需要回滚。
- 更换 Windows/macOS 电脑作为新主机。

不适用情况：

- 多台电脑离线分别录入后自动合并。
- 多主机同时写入。
- 云端同步恢复。

## 2. 日常准备

管理员需要确认：

1. 主机或单机电脑使用“系统设置”中的备份策略。
2. 第二备份目录已设置到主机之外的位置，例如移动硬盘、NAS、局域网共享目录或另一台电脑共享目录。
3. 首页或系统设置中“第二备份”显示为可用。
4. 定时备份处于开启状态。默认每 6 小时检查一次是否需要创建 `auto_interval` 备份。
5. 至少每周手动抽查一次第二备份目录，确认能看到 `aster-backup-YYYYMMDD-HHMMSS-短ID.zip` 文件。

## 3. 自动备份和保留策略

系统会创建以下备份：

- `auto_startup`：每天首次启动自动备份一次。
- `auto_interval`：应用运行中按定时间隔自动备份，默认间隔 6 小时。
- `manual`：管理员手动创建。
- `before_import`：Excel 正式导入前自动创建。
- `before_restore`：恢复备份前自动创建。
- `before_migration`：数据库迁移前预留。

自动备份保留策略：

- 最近 7 天：保留全部自动备份。
- 最近 8 到 30 天：每天保留最后一个自动备份。
- 最近 12 个月：每月保留最后一个自动备份。
- 超过 12 个月的自动备份自动清理。
- 手动备份、导入前备份和恢复前保护备份默认不自动删除。

每次清理自动备份都会写入操作日志。

## 4. 主机损坏后的切换流程

操作人：管理员。

目标：在 30 分钟内把另一台电脑恢复为新主机。

步骤：

1. 找到第二备份目录中的最新可用备份包。
2. 在新电脑安装并启动 Aster。
3. 使用管理员账号登录。
4. 进入“系统设置”。
5. 在“恢复备份”中输入备份包完整路径。
6. 点击“校验”，确认备份包结构、Schema、来源主机、来源系统和 SHA256 校验通过；如果修改备份文件路径，必须重新校验。
7. 在确认文本中输入 `RESTORE`。
8. 点击“恢复”。
9. 等待系统完成恢复前保护备份、替换数据库和健康检查。
10. 将运行模式设置为“主机模式”。
11. 启动主机服务，记录新的主机 IP、端口和配对码。
12. 其他电脑进入“系统设置”，通过“发现”或手动填写新主机地址重新连接。
13. 客户端完成配对后，检查首页统计、库存台账和最近流水。

## 5. 恢复失败处理

如果恢复失败：

1. 不要删除原备份包。
2. 记录错误提示。
3. 优先换用同一天更早的备份包重新校验。
4. 如果恢复已经替换数据库但健康检查失败，系统会自动回滚到 `before_restore` 保护备份。
5. 如果自动回滚失败，按错误提示中的 `before_restore` 保护备份路径手工恢复。
6. 恢复成功后必须检查数据库健康状态、最近备份时间和第二备份状态。

## 6. 跨平台恢复注意事项

- Windows 创建的备份可以在 macOS 恢复。
- macOS 创建的备份可以在 Windows 恢复。
- 恢复后本机数据目录由当前系统重新计算，不沿用旧电脑绝对路径。
- 恢复后默认导出目录和默认备份目录会重置到新主机应用目录，第二备份目录需要在新主机上重新配置。
- Excel 导入导出和备份恢复不依赖 Microsoft Excel 或 WPS。
- Windows 作为主机时需要检查防火墙是否允许 Aster 或默认端口 `17871` 的局域网入站连接。
- macOS 作为主机时需要允许 Aster 进行局域网访问。
- 客户端手动填写主机地址时只填 IP 或主机名，不填写 `http://`、端口或路径；端口默认 `17871`，有效范围为 `1024-65535`。

## 7. 验收检查清单

- 可以创建手动备份。
- 备份包中包含 `aster.sqlite`、`backup-meta.json`、`app-settings.json` 和 `import-reports/`。
- 备份记录和恢复预览能看到来源主机、来源系统、数据库大小和 SHA256。
- 第二备份目录可写，并能看到复制出的备份包。
- 启动自动备份和运行中定时备份可生成记录。
- 自动备份按保留策略清理，手动备份不会被自动删除。
- 恢复前会自动生成 `before_restore` 保护备份。
- 恢复后 SQLite `integrity_check` 通过。
- 新主机能被客户端发现、连接和配对。
- 客户端连接新主机后看到同一套库存和报表数据。
