# Aster 安全与工程化整改实施计划

## 实施状态

- 阶段一至四及阶段六已完成：工程门禁、系统凭据库、强制改密、设备/用户会话分离、TLS、OPAQUE、资源限制和全量验证均已落地。
- 阶段五已完成：所有返回集合的远程路由均在 repository SQL 层分页，客户端自动聚合；同时完成写入 2,000 行限制、8 MiB 响应上限、SQLite 30 秒截止时间和断开取消。

## 交付原则

- 先建立可执行门禁，再迁移安全边界，任何阶段都保持可构建、可测试。
- 新模块遵守设计文档的分层和端口适配器约束；遗留巨型文件只减不增。
- 每个安全缺陷先有回归测试，再切换生产路径。
- 旧明文协议不保留业务兼容；业务数据库保留，客户端升级后重新配对。

## 阶段一：工程门禁与基线

1. 记录现有巨型文件、生产 `unwrap/expect/panic` 和跨层依赖基线。
2. 新增架构检查脚本，限制新文件/函数规模、禁止危险新增模式并检查前端分层导入。
3. 修复现有严格 Clippy 错误，加入 CI。
4. 建立前端测试与 lint 入口；安全模块必须纳入测试。

## 阶段二：凭据与默认密码

1. 新增 `application/ports` 凭据接口及系统凭据库适配器。
2. 新增窄 Tauri command，前端认证模块通过 command 读写凭据。
3. 启动时删除旧 `localStorage` 明文密码；不迁移旧密码。
4. 数据库迁移增加 `must_change_password`，仅对实际仍匹配默认密码的管理员标记。
5. service 层增加强制改密门禁，前端增加不可跳过的改密流程。
6. 密码校验移出 SQLite 全局锁，并增加登录失败限流。

## 阶段三：设备与用户会话

1. 增加设备、配对注册、用户会话和安全限流表。
2. 设备 token 和会话 token 只保存 SHA-256 哈希。
3. 登录签发绑定设备的会话；所有远程授权从会话加载用户，删除用户 ID 请求头信任。
4. 实现闲置 8 小时、绝对 24 小时、主动吊销、密码变化吊销和定时清理。
5. 本机/主机 Tauri command 与远程路由共享授权和强制改密规则。

## 阶段四：TLS 与 OPAQUE 配对

1. 用有界异步 HTTP/TLS 服务替换裸 TCP 每连接线程模型。
2. 生成并持久化主机证书身份，客户端固定证书指纹。
3. 使用 RFC 9807 `opaque-ke` 实现一次性 12 位配对，transcript 绑定证书、设备和协议版本。
4. 新版本关闭旧明文业务端点，清除旧客户端 token 并引导重新配对。
5. 加入超时、并发、正文大小、错误状态和敏感日志过滤。

## 阶段五：分页与容量保护

1. [已完成] 定义通用分页 DTO、scope 绑定偏移游标和库存键集游标；游标使用 HMAC-SHA256 防篡改，并绑定查询条件、数据库实例和结果集修订号。
2. [已完成] 所有返回集合与报表 section 的 repository 查询均增加稳定排序和 SQL 分页。
3. [已完成] remote helper 自动聚合分页，保持现有 React 使用方式。
4. [已完成] 写请求限制 2,000 行；数据库任务绑定 30 秒截止时间和断开取消令牌。
5. [已完成] 验证库存键集分页、1,005 行主数据三页聚合、501 行报表跨页、游标 scope 隔离、数据变化失效、HMAC 篡改拒绝和客户端心跳隔离。

## 阶段六：全量验证

1. 运行前端类型、lint、测试和生产构建。
2. 运行 Rust 格式、严格 Clippy、测试和依赖审计。
3. 运行架构、迁移、旧协议拒绝、MITM/重放、凭据隔离和容量测试。
4. 更新用户手册、升级说明和灾难恢复文档。
5. 确认工作树仅包含本次变更并记录仍需人工跨平台验收的项目。
