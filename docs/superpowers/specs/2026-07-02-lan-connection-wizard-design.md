# 局域网连接向导设计

日期：2026-07-02

## 背景

当前“局域网主机/客户端”流程暴露了过多工程概念。用户需要理解运行模式、主机服务、IP、端口、发现、保存连接、测试连接、配对码等多个步骤，还要在设置页和两个独立窗口之间切换。实际业务用户只关心两件事：

- 哪台电脑保存正式数据。
- 其他电脑如何连上它一起用。

因此需要把现有流程收敛为一个连接向导，降低首次配置和日常维护成本。

## 目标

- 用“主电脑 / 其他电脑”替代“主机 / 客户端”的主要用户表达。
- 设置页只保留一个清晰入口：`多电脑连接`。
- 用户选择“这台作为主电脑”后，一键完成模式切换和主机服务启动。
- 用户选择“连接到主电脑”后，自动搜索局域网主电脑，用户只需要选择主电脑并输入 6 位配对码。
- 后台自动串联保存地址、测试连接、切换客户端模式、完成配对、刷新业务状态。
- 保留现有 Tauri/Rust 局域网同步架构和 API，优先重做体验，不扩大底层风险。

## 非目标

- 不引入云端数据库。
- 不改变现有本地 SQLite 作为主库的架构。
- 不在本轮实现跨公网或异地同步。
- 不改变现有 token 鉴权和配对码机制，只封装到向导流程里。
- 不移除高级配置能力；端口/IP 手动输入仍作为“手动连接”兜底。

## 信息架构

### 设置页

把原来的“局域网主机/客户端”设置块改成“多电脑连接”。

显示内容：

- 当前状态：
  - `单机使用`
  - `这台是主电脑`
  - `已连接到主电脑`
  - `连接异常`
- 当前连接摘要：
  - 主电脑名称或地址
  - 端口仅在高级详情中显示
  - 已连接设备数量
  - 最近检测时间
- 主按钮：`打开连接向导`
- 次按钮：
  - 主电脑状态下：`查看已连接设备`
  - 其他电脑状态下：`重新连接`

不再在设置页直接显示复杂的 IP、端口、配对入口。

### 连接向导窗口

入口标题：`多电脑连接`

首屏两个选择：

- `这台作为主电脑`
  - 说明：正式数据保存在这台电脑，其他电脑连接过来一起使用。
- `连接到主电脑`
  - 说明：这台电脑不保存正式数据，连接到已有主电脑使用同一套库存数据。

## 流程设计

### 流程一：这台作为主电脑

1. 用户点击 `这台作为主电脑`。
2. 系统提示确认：
   - 标题：`把这台电脑设为主电脑？`
   - 文案：`正式库存数据将保存在这台电脑。其他电脑需要连接这台电脑后才能共同使用。`
3. 用户点击 `开启共享`。
4. 后台执行：
   - 设置 `runtime_mode=host`
   - 启动主机服务
   - 获取主机服务状态
   - 刷新客户端列表
5. 成功页显示：
   - `主电脑已开启`
   - 大号 6 位配对码
   - 本机可连接地址，作为辅助信息
   - 已连接设备列表
   - 按钮：`刷新配对码`、`完成`

配对码区域应突出显示，便于用户口头告诉另一台电脑。

### 流程二：连接到主电脑

1. 用户点击 `连接到主电脑`。
2. 系统自动执行主机发现，展示状态：
   - `正在搜索局域网内的主电脑...`
3. 找到主电脑后展示列表：
   - 主电脑名称或地址
   - Aster 版本
   - 数据库 schema 版本
   - 健康状态
4. 用户选择一台主电脑。
5. 用户输入主电脑屏幕上的 6 位配对码。
6. 用户点击 `连接`。
7. 后台串联执行：
   - 保存主机地址和端口
   - 切换 `runtime_mode=client`
   - 测试主机连接
   - 使用配对码调用 `pair_with_host`
   - 保存 token
   - 刷新应用状态和业务数据
8. 成功页显示：
   - `已连接到主电脑`
   - 主电脑地址
   - 最近检测时间
   - 按钮：`完成`

### 手动连接兜底

如果自动发现不到主电脑，显示：

- 文案：`没有找到主电脑。请确认主电脑已开启共享，并且两台电脑在同一个局域网。`
- 按钮：
  - `重新搜索`
  - `手动输入地址`

手动输入地址时只显示：

- 主电脑地址
- 高级展开项：端口，默认 `17871`
- 按钮：`测试并继续`

用户不需要先保存，再打开另一个窗口配对；测试成功后直接进入配对码输入。

## 状态与错误处理

### 主电脑启动失败

常见原因：

- 端口被占用
- 防火墙拦截
- 没有权限监听

用户文案：

`主电脑共享没有启动成功。请关闭占用端口的程序，或允许 Aster 通过系统防火墙后重试。`

提供：

- `重试`
- `查看高级信息`

高级信息显示原始错误。

### 找不到主电脑

用户文案：

`没有找到主电脑。请确认主电脑已经打开“主电脑共享”，并且两台电脑连接的是同一个 Wi-Fi 或局域网。`

提供：

- `重新搜索`
- `手动输入地址`

### 配对码错误

用户文案：

`配对码不正确或已刷新，请输入主电脑当前显示的 6 位数字。`

保留当前选中的主电脑，不清空用户已经选择的主机。

### 已配对但连接异常

设置页状态显示 `连接异常`，业务操作继续按现有逻辑暂停。

提供：

- `重新检测`
- `重新连接`
- `打开连接向导`

重新连接优先使用已保存的主机地址和 token 测试连接；只有 token 无效时才要求重新输入配对码。

## 数据与接口

优先复用现有命令：

- `start_host_service`
- `get_host_service_status`
- `list_client_connections`
- `discover_hosts`
- `test_host_connection`
- `save_client_config`
- `pair_with_host`

建议新增一个前端聚合流程，而不是先新增 Rust 大接口。原因是现有命令边界清晰，向导主要是编排体验。

如后续发现前端编排错误处理过散，可再新增 Rust 命令：

- `connect_to_discovered_host`
- `enable_host_mode_and_start_service`

但第一阶段不强制。

## UI 组件建议

新增或改造组件：

- `ConnectionWizard`
- `ConnectionRoleStep`
- `EnableHostStep`
- `DiscoverHostStep`
- `PairHostStep`
- `ConnectionSuccessStep`
- `ConnectionTroubleshooting`

现有组件可合并：

- `ClientConnectionEditor`
- `ClientPairingEditor`

这两个窗口后续可以作为向导内部步骤保留部分表单逻辑，但不再作为设置页的主要入口。

## 文案规范

主要面向用户使用：

- 使用 `主电脑`，不使用 `主机`
- 使用 `其他电脑`，不使用 `客户端`
- 使用 `开启共享`，不使用 `启动主机服务`
- 使用 `连接到主电脑`，不使用 `配置客户端连接`
- 使用 `配对码` 可以保留，因为用户能理解

高级详情中可以保留：

- IP 地址
- 端口
- schema 版本
- token 错误

## 验收标准

### 主电脑路径

- 从设置页点击 `打开连接向导`。
- 选择 `这台作为主电脑`。
- 点击 `开启共享` 后，应用自动切换到主电脑模式并启动服务。
- 成功页显示 6 位配对码。
- 设置页状态显示 `这台是主电脑`。
- 已连接设备数量可刷新显示。

### 其他电脑路径

- 从设置页点击 `打开连接向导`。
- 选择 `连接到主电脑`。
- 自动搜索能发现局域网主电脑。
- 选择主电脑后输入配对码。
- 点击 `连接` 后，不需要再手动保存地址或打开第二个配对窗口。
- 成功后设置页状态显示 `已连接到主电脑`。
- 业务页面能读取主电脑数据并提交业务单据。

### 兜底路径

- 自动发现失败时能手动输入地址。
- 手动输入地址后可测试并继续配对。
- 配对码错误时错误提示清楚，且保留已选主电脑。
- 主电脑关闭后，其他电脑显示连接异常；主电脑恢复后可重新检测恢复。

## 测试建议

- React 单元测试或组件测试覆盖向导状态流转。
- Rust 现有主机服务测试继续保留。
- 增加前端命令 mock 测试：
  - 主电脑开启成功
  - 发现主电脑成功
  - 发现为空后手动连接
  - 配对码错误
  - token 已存在时重新检测
- 手工验收覆盖：
  - Windows 主电脑 + macOS 其他电脑
  - macOS 主电脑 + Windows 其他电脑

## 迁移策略

- 不删除旧设置数据。
- 已经处于 host/client 的用户打开新设置页时，按现有 `runtime_mode` 显示对应状态。
- 已保存主机地址和 token 的其他电脑，直接显示 `已连接到主电脑` 或 `连接异常`。
- 保留旧命令，避免影响远程业务读写。

## 风险

- UDP 自动发现可能被防火墙拦截，因此必须保留手动输入地址。
- Windows 防火墙可能阻止入站连接，主电脑启动成功不代表其他电脑一定能连上，需要在错误文案里引导。
- 当前 `discover_hosts` 依赖默认端口，用户改端口后发现体验会变差；第一阶段建议隐藏端口但保留高级设置。
- 向导聚合多个异步命令，前端需要清晰处理 loading、取消和重复点击。
