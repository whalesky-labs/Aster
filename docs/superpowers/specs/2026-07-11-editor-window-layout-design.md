# 子窗口公共布局修复设计

## 问题

macOS 动态子窗口没有继承主窗口的 Overlay 标题栏设置，WebView 被原生标题栏向下偏移后仍按外框高度裁切，底部会露出一个标题栏高度的窗口底色。当前子窗口壳还使用 `100vh`，操作栏又使用窗口级 `position: fixed`，进一步混用了原生外框、WebView 和 CSS viewport 三套定位基准。

## 目标

- 所有编辑、设置、连接、库存和详情子窗口完整填满 WebView 可视区域。
- 表单内容独立滚动，底部操作栏始终位于子窗口内容区底部。
- 详情和长表格窗口保持现有滚动行为。
- 不修改字段、窗口标题、业务逻辑、Tauri 命令和默认窗口尺寸。

## 设计

1. 动态子窗口的原生窗口背景色与当前浅色或深色主题的内容背景同步，消除 macOS 原生内容边界露出的色差；Windows 保持相同行为。
2. 子窗口页面使用统一的 `.editor-shell` 作为四边贴合的根布局，尺寸来自包含块而不是重复推算原生窗口高度。
3. 普通 `.editor-form` 使用 `minmax(0, 1fr) auto` 两行网格；库存单据、库存调整、盘点录入、只读详情和软件更新等 `.editor-actions` 直属 `.editor-document` 的窗口，在各自现有内容行后追加 `auto` 操作栏行。
4. `.editor-actions` 回归所属网格的文档流，不再以 viewport 为基准固定定位。没有操作栏的只读窗口继续让内容占满可用空间。
5. 仅删除动作栏避让值：调整 `.editor-form-grid`、`.editor-table-scroll`、`.readonly-editor-scroll`、`.stocktake-detail-scroll` 和 `.software-update-scroll` 的多余底部预留，同时保留正常内容内边距、表格横向滚动、详情嵌套滚动和 sticky 表头。
6. `html`、`body`、`#root` 和编辑窗口 body 保持完整高度链，并阻止外层滚动。

## 验证

- 生产构建与工程规范检查通过。
- 检查预算、基础资料、用户、设置、连接、库存单据、库存调整、只读详情、软件更新、盘点详情和盘点录入等不同高度的子窗口。
- 短内容窗口的操作栏贴底且不出现额外黑边；长内容窗口只有预定内容区域纵向滚动。
- 宽表格仍可横向滚动，sticky 表头保持吸附，滚动至末行或末字段时不被操作栏遮挡。
- 只读详情的外层和嵌套滚动区域均可到达末尾。
- 主窗口布局不受影响。
