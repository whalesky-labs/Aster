# 子窗口公共布局修复设计

## 问题

macOS 子窗口使用原生标题栏时，WebView 的内容高度与窗口外框高度不同。当前子窗口壳使用 `100vh`，操作栏又使用窗口级 `position: fixed`，两套高度和定位基准混用，导致表单底部与实际内容区不能稳定贴合，所有子窗口都可能露出额外窗口底色。

## 目标

- 所有编辑、设置、连接、库存和详情子窗口完整填满 WebView 可视区域。
- 表单内容独立滚动，底部操作栏始终位于子窗口内容区底部。
- 详情和长表格窗口保持现有滚动行为。
- 不修改字段、窗口标题、业务逻辑、Tauri 命令和默认窗口尺寸。

## 设计

1. 子窗口页面使用统一的 `.editor-shell` 作为四边贴合的根布局，尺寸来自包含块而不是重复推算原生窗口高度。
2. `.editor-form` 使用两行网格：第一行为可滚动内容，第二行为操作栏。
3. `.editor-actions` 回归网格文档流，不再以 viewport 为基准固定定位。
4. 删除为固定操作栏预留的底部空白；表单和表格只保留正常内容内边距。
5. `html`、`body`、`#root` 和编辑窗口 body 保持完整高度链，并阻止外层滚动。

## 验证

- 生产构建与工程规范检查通过。
- 检查预算、基础资料、用户、设置、连接、库存单据和盘点等不同高度的子窗口。
- 缩放子窗口时，内容可滚动、操作栏贴底且不出现额外黑边。
- 主窗口布局不受影响。
