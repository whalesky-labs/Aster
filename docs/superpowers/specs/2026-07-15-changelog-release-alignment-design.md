# Aster 中文更新日志与线上发布对齐设计

日期：2026-07-15

## 目标

让 `CHANGELOG.zh-CN.md` 与 GitHub 最新正式 Release 保持一致，并为尚未发布的当前改动建立新的 `[Unreleased]` 记录。

## 对齐规则

- GitHub Release `v0.1.10`（2026-07-11）是最新正式发布基准。
- 当前 `[Unreleased]` 中与 `v0.1.10` 线上功能正文一致的全部分类和条目，原样移动到 `[0.1.10] - 2026-07-11`。
- Release 自动追加的下载表不复制进仓库更新日志，保持现有历史版本格式和发布脚本职责不变。
- `0.1.9` 及更早历史内容不改写。

## 新增日志范围

新的 `[Unreleased]` 按“新增 / 优化 / 修复”组织，覆盖 `v0.1.10` 后已经完成但尚未发布的用户可见或交付相关变化：

- 系统安全凭证保存与旧明文凭证迁移；
- 子窗口主题、底部布局、置前能力和错误反馈；
- 列表页面统一的“页面功能 → 搜索 → 表格”结构；
- 物品档案供应商下拉筛选及筛选后导出；
- 仅管理员可用的全量库存台账 Excel 导出；
- Windows 主窗口系统托盘；
- 后端真实分页、SQLite 查询优化、定向刷新、登录资源按需加载；
- 前端测试、浏览器布局测试、Windows CI 与 500 行源码约束。

只记录已经实现并经过验证的内容，不预写未完成能力，不把纯文件移动逐项暴露给最终用户。

## 验证

- 对比 GitHub API 返回的 `v0.1.10` 功能正文与本地对应版本段；比对时排除发布流程自动追加的下载表。
- 运行发布说明生成脚本，确认能够从新 `[Unreleased]` 读取下一版本内容。
- 执行工程规范与 `git diff --check`，确认格式和版本顺序正确。
