# Aster 项目复审整改设计

日期：2026-07-15

## 目标

一次性处理 2026-07-15 全项目 review 中确认的运行时权限、敏感凭证、日期正确性、事务边界、自动化测试、跨平台 CI、数据加载性能、SQLite 查询、源码规模和登录资源问题。保持现有业务流程、Tauri command 名称、远程协议语义和 SQLite 数据兼容性。

## 设计边界

- 登录密码、SMTP 密码和客户端配对 token 均通过系统安全凭证保存；SQLite、设置快照、备份包和前端状态不再包含明文秘密。
- 已存在的 SQLite 明文秘密与旧 Keychain 命名空间在启动时安全迁移，迁移完成后删除旧值。
- 子窗口继续统一由 `openEditorWindow` 创建；补齐置前权限，并把创建失败反馈给主界面。
- 月份和日期默认值统一由本地时区工具生成，不再使用 UTC `toISOString()` 截取月份。
- 密码重置、系统设置和配对配置的多步写入使用事务；SMTP 网络 I/O 不持有 SQLite 全局锁。
- 前端增加纯逻辑单测和编辑窗口浏览器布局回归测试；普通 CI 增加 Windows 编译检查。
- 页面刷新由“全量刷新所有模块”改为按业务域失效；高数据量列表逐步使用后端分页结果，前端不再先聚合全部记录再切页。
- 月度查询改为可使用索引的日期范围，增加与筛选/排序一致的复合索引。
- 所有生产源码控制在 500 行以内，移除工程基线中的大文件豁免。
- 登录动画资源改为按需加载，减少启动 JS 和图片解码压力。

## 兼容与失败策略

- 安全凭证迁移采用“写入新凭证成功后再删除旧值”；失败时保留旧值并返回明确错误。
- 旧备份恢复后删除其中的 SMTP 密码和配对 token，要求用户重新输入或重新配对。
- 数据库迁移只增加索引和非敏感状态字段，不改变已有业务记录含义。
- 子窗口错误通过应用级事件显示在现有错误区域，不使用静默失败。

## 验证

- TypeScript 构建、工程规范、Vitest、Playwright 编辑窗口布局测试。
- Rust fmt、Clippy、全部 Rust 测试和新增安全迁移/事务/查询测试。
- macOS Tauri debug bundle 实机验证主题、窗口置前和底部布局。
- Windows GitHub Actions 执行前端构建与 Rust check。
- SQLite `EXPLAIN QUERY PLAN` 确认月度流水、单据、审计日志和备份查询使用目标索引。
